## 1. 产品概述

### 1.1 产品定位

一款**免登录、免签名**的抖音视频下载工具：用户粘贴抖音 App 的分享文案（含 `v.douyin.com` 短链），工具自动解析并下载**无水印**的 MP4 原片到本地。

### 1.2 背景

- 抖音 App 内保存的视频带有官方水印（账号 ID + logo 动态角标）。
- 抖音网页版官方 API（`/aweme/v1/web/aweme/detail`）需要计算 `X-Bogus` / `a_bogus` 反爬签名并携带有效 Cookie，实现和维护成本高。
- 分享页（`iesdouyin.com/share/video/`）是抖音为「未安装 App 的用户」提供的 H5 落地页，其 HTML 内嵌了完整的视频元数据 JSON，**不需要任何签名**，是目前最稳定的免签方案。

### 1.3 目标用户与场景

| 场景 | 描述 |
|---|---|
| 个人收藏 | 保存自己喜欢的视频原片，防止作品被删除后无法回看 |
| 素材归档 | 创作者备份自己发布的作品原片 |
| 二次剪辑 | 在获得授权的前提下获取无水印素材 |

### 1.4 核心价值

1. **零门槛**：无需登录抖音账号、无需 App、无需浏览器插件。
2. **无水印**：直接获取 CDN 原片，画面干净。
3. **轻依赖**：全流程仅需 HTTP 请求（curl / requests 即可实现），无签名计算。

## 2. 核心下载逻辑（技术方案）

整个下载流程分为 **4 步**，已用真实分享链接实测验证通过。

### 流程总览

```
分享文案
   │  正则提取短链
   ▼
① v.douyin.com/xxxx ──302──▶ 拿到视频 ID
   │
   ▼
② iesdouyin.com/share/video/{ID}  （移动端 UA）
   │  解析 HTML 内嵌的 window._ROUTER_DATA
   ▼
③ 得到 playwm 地址 ──替换──▶ play（去水印）
   │
   ▼
④ aweme.snssdk.com/aweme/v1/play/ ──302──▶ CDN 无水印 MP4
```

### 步骤 ①：从分享文案提取并解析短链

分享文案格式示例：

```
3.89 复制打开抖音，看看【xxx的作品】断头之站 # 现场直击 ...
https://v.douyin.com/i5IeTINBeZQ/ XzT:/ 08/15 C@H.Ii :6pm
```

- 用正则 `https://v\.douyin\.com/[\w-]+/?` 提取短链。
- 对短链发起请求（**不跟随重定向**），读取 302 响应的 `Location` 头：

```
Location: https://www.iesdouyin.com/share/video/7663108625456311626/?...
```

- 用正则 `/share/video/(\d+)` 提取**视频 ID**（也叫 aweme_id / item_id）。

### 步骤 ②：抓取分享页，解析内嵌 JSON

请求 `https://www.iesdouyin.com/share/video/{视频ID}/`，**必须使用移动端 User-Agent**（桌面 UA 会被重定向到验证页/首页）：

```
Mozilla/5.0 (iPhone; CPU iPhone OS 17_2 like Mac OS X) AppleWebKit/605.1.15
(KHTML, like Gecko) Version/17.2 Mobile/15E148 Safari/604.1
```

返回的 HTML（约 38 KB）中嵌有一段 SSR 数据：

```html
<script>window._ROUTER_DATA = { ... }</script>
```

关键字段路径：

```
loaderData
 └─ "video_(id)/page"
     └─ videoInfoRes
         └─ item_list[0]
             ├─ desc                     # 视频标题/文案
             ├─ author.nickname          # 作者昵称
             └─ video
                 ├─ play_addr.uri        # 视频资源 ID（vid）
                 ├─ play_addr.url_list[] # 播放地址（带水印 playwm）
                 ├─ duration             # 时长（毫秒）
                 └─ cover.url_list[]     # 封面图
```

实测拿到的播放地址：

```
https://aweme.snssdk.com/aweme/v1/playwm/?video_id=v0d00fg10000d9cd9b27dld66v76poe0&ratio=720p&line=0
```

### 步骤 ③：去水印 —— `playwm` → `play`

`playwm` 中的 **wm = watermark（水印）**。将 URL 路径里的 `playwm` 替换为 `play`，同一接口即返回无水印源片的重定向。

两者 302 目标的差异可直接验证：

| 接口 | CDN 路径特征 | 说明 |
|---|---|---|
| `/aweme/v1/playwm/` | `.../video/cn/mps/logo/r/p/...` | 路径含 `logo`，服务端烧录了水印 |
| `/aweme/v1/play/` | `.../video/tos/cn/tos-cn-ve-15/...` | 无 `logo`，原片直出 |

### 步骤 ④：跟随重定向下载

对 `play` 接口发起请求并跟随 302（同样带移动端 UA），即可下载 MP4：

```bash
curl -sL 'https://aweme.snssdk.com/aweme/v1/play/?video_id={vid}&ratio=720p&line=0' \
  -A '<移动端UA>' -o output.mp4
```

**实测结果**（本次验证的视频《断头之站》）：

| 项 | 值 |
|---|---|
| 文件大小 | 6.9 MB |
| 分辨率 | 1280 × 720 |
| 编码 | H.264 + AAC |
| 时长 | 22.4 s（与页面元数据一致） |
| 水印 | 无 |

## 3. 完整参考实现（Shell 版）

```bash
#!/bin/bash
# 用法: ./douyin_dl.sh "分享文案或短链"
UA='Mozilla/5.0 (iPhone; CPU iPhone OS 17_2 like Mac OS X) AppleWebKit/605.1.15 (KHTML, like Gecko) Version/17.2 Mobile/15E148 Safari/604.1'

# ① 提取短链并解析视频 ID
SHORT=$(echo "$1" | grep -oE 'https://v\.douyin\.com/[[:alnum:]-]+/?')
LOC=$(curl -s -o /dev/null -w '%{redirect_url}' "$SHORT" -A "$UA")
VID_ID=$(echo "$LOC" | grep -oE '/share/video/[0-9]+' | grep -oE '[0-9]+')

# ② 抓分享页，提取 video_id（资源 vid）
VID=$(curl -s "https://www.iesdouyin.com/share/video/$VID_ID/" -A "$UA" \
  | grep -oE 'video_id=[^&"]+' | head -1 | cut -d= -f2)

# ③④ 用 play（无水印）接口下载
curl -sL "https://aweme.snssdk.com/aweme/v1/play/?video_id=$VID&ratio=720p&line=0" \
  -A "$UA" -o "douyin_$VID_ID.mp4"
```

## 5. 非功能需求与约束

| 类别 | 约束 |
|---|---|
| **UA 要求** | 步骤②④必须使用移动端 UA，否则被跳转到验证页 |
| **链接时效** | CDN 最终地址带签名参数，**数小时内有效**，必须即取即下，不可缓存分发 |
| **清晰度上限** | 分享页只暴露 720P 地址；更高清晰度需签名 API |
| **频控** | 无签名接口对高频请求有风控，建议单 IP 请求间隔 ≥ 1s，失败退避重试 |
| **不落地敏感数据** | 不需要也不应存储任何 Cookie / 账号信息 |

## 6. 边界情况处理

| 场景 | 表现 | 处理策略 |
|---|---|---|
| 图集（图文作品） | 重定向到 `/share/note/` 而非 `/share/video/` | 识别 note 路径，改走 `images[]` 下载图片 |
| 视频已删除 / 私密 | `_ROUTER_DATA` 中 `item_list` 为空或含 `filter_detail` | 报错「视频不存在或不可见」 |
| 直播分享链接 | 重定向到 `/share/live/` | 明确提示不支持直播 |
| 短链失效 | 302 到抖音首页而非 share 页 | 报错「链接已失效」 |
| 文案含多个链接 | 正则可能匹配多条 | 取第一条，或全部入队 |

## 7. 风险与合规

### 7.1 技术风险

| 风险 | 等级 | 说明与预案 |
|---|---|---|
| 分享页结构变更 | 中 | `_ROUTER_DATA` 的 JSON 路径历史上变动过；解析层做成独立模块，用「递归找 `item_list` 键」的容错策略（本次实现即采用此法） |

## 8. 反爬机制深度分析与代理防封

### 8.1 抖音的六层限制机制

| # | 机制 | 说明 | 本项目对策 |
|---|---|---|---|
| 1 | **IP 频控** | 同一 IP 高频请求 → 返回空数据 / 滑块验证 / 临时封禁 | **代理池轮换**（核心）+ 请求分散 |
| 2 | **UA 校验** | 分享页要求移动端 UA，桌面 UA 跳验证页 | 移动端 **UA 池轮换**（4 个 iOS/Android UA） |
| 3 | **防盗链 Referer** | 部分 CDN 资源校验 Referer | 出站统一带 `Referer: https://www.douyin.com/` |
| 4 | **URL 签名时效** | `play_addr`/图片 URL 带 `x-expires`+`x-signature`，几小时失效 | 无法延长，**即取即用**；后端流式代理避免地址暴露给前端缓存 |
| 5 | **ttwid / msToken Cookie** | 网页版高清源与 detail API 需要 | 免签的 H5 分享页方案**绕过**，无需 Cookie |
| 6 | **a_bogus / X-Bogus 签名** | Web API 需前端 JS 加密参数 | H5 分享页**不需要签名**，从根上规避 |

**结论**：真正会导致「批量下载被封」的是第 1 层 IP 频控。因此代理池是防封的关键手段，其余各层本方案已从架构上规避。

### 8.2 代理池设计

参考成熟开源方案 [jhao104/proxy_pool](https://github.com/jhao104/proxy_pool)（健康检查+评分淘汰）与 [yukkcat/socks5-proxy](https://github.com/yukkcat/socks5-proxy)（自动轮换+故障转移+仪表盘）的设计。

```
每个出站请求（短链解析 / 分享页 / 封面 / 视频流 / 图集）
        │
        ▼
  candidates()：按轮换策略排序启用中的代理
        │  依次尝试（最多 retries 个）
        ├─ 连接失败(URLError/超时) → fail+1，自动转移到下一个代理
        │                            fail≥阈值 → 自动禁用(auto_off)
        ├─ 源站返回 4xx/5xx        → 代理是通的，不惩罚，直接上抛
        └─ 成功                    → fail 归零，记录延迟，返回
        │
    全部代理失败
        ├─ force_proxy=on  → 报错，拒绝直连（防真实 IP 暴露）
        └─ force_proxy=off → 降级直连
```

**多格式解析**（`ProxyManager.parse_proxy`）——代理商导出格式五花八门，统一规范成 `scheme://[user:pass@]host:port`：

| 输入格式 | 解析结果（默认协议 socks5） |
|---|---|
| `socks5://user:pass@1.2.3.4:1080` | 原样 |
| `1.2.3.4:1080:user:pass` | `socks5://user:pass@1.2.3.4:1080` |
| `user:pass@1.2.3.4:8080` | `socks5://user:pass@1.2.3.4:8080` |
| `1.2.3.4:1080` | `socks5://1.2.3.4:1080` |
| `http://1.2.3.4:3128` | 原样 |

- **协议支持**：`http`/`https`（urllib 原生 CONNECT）、`socks5`/`socks5h`/`socks4`/`socks4a`（PySocks，`h`/`a` 后缀走远端 DNS 解析防泄露）。密码含特殊字符自动 URL 编码。
- **默认协议可配**：无前缀的裸地址按 `default_protocol` 解析，默认 **socks5**（代理多为 socks5）。
- **轮换策略**：`round_robin`（轮询+健康优先）/ `random`（随机）/ `least_fail`（最少失败优先），可切换。
- **自动重试**：单请求失败自动换下一个代理，最多试 `retries` 个（默认 3）。
- **故障区分**：连接层错误才判代理故障；源站 4xx/5xx 不冤枉代理（如签名过期返回 403）。
- **强制代理**：默认开启，代理全挂宁可报错也不用真实 IP 直连。
- **持久化**：代理列表与设置存 `data/config.json`，Docker 挂载 `/data` 卷保留。

### 8.3 后台健康检查与自愈

- **守护线程**：按 `health_interval_min`（默认 10 分钟）并发测试所有启用中（及被自动禁用）的代理。
- **测试内容**：出口 IP + 延迟（api.ipify.org）+ **抖音可达性**（www.iesdouyin.com，确认代理能真正连上抖音，而非仅通外网）。
- **自动禁用**：连续失败达 `auto_disable_fail`（默认 5）→ 标记 `auto_off` 并停用，退出轮换。
- **自愈**：被自动禁用的代理在后续健康检查中恢复可用 → 自动重新启用；用户手动禁用的不会被自愈干扰。

### 8.4 管理后台（隐藏入口 `/admin_d`，首页不暴露）

- `ADMIN_PASSWORD` 密码登录，服务端会话（HttpOnly Cookie，12h）；`secrets.compare_digest` 防时序攻击。
- **代理管理**：多格式批量添加、**预览解析**（入库前看规范化结果）、启停、删除、单个/批量并发测速（出口 IP + 延迟 + 抖音可达）。
- **策略配置**：强制代理、轮换策略、每请求重试数、默认协议、自动健康检查开关与间隔、自动禁用阈值、抖音可达检测开关——全部即时保存。
- **统计**：代理总数/启用数、抖音可达数、经代理/直连计数、自动重试次数。
- **安全**：媒体代理保留域名白名单（仅抖音系 CDN）防 SSRF。

### 8.5 进一步可加的过限制手段（迭代方向）

| 手段 | 收益 | 成本 |
|---|---|---|
| 请求间隔 / 令牌桶限速 | 降低单代理触发频控概率 | 低 |
| Cookie（ttwid）池 | 提升成功率、解锁部分高清 | 中，需维护 Cookie |
| 住宅/移动代理替代数据中心 IP | 大幅降低被识别为爬虫 | 高（付费代理） |
| 代理来源自动抓取 + 入库 | 免费代理自动补充池子 | 中，质量参差 |
| 接入带签名的 Web API | 拿 1080P+ 更高清晰度 | 高，需实现 a_bogus 签名 |

### 代理链路实测

用本地测试代理 `tools/testproxy.py`（打印每条经过的请求）验证「所有出站走代理」：解析一个视频后，代理日志逐行记录了全部 4 个上游 + 测速：

```
[proxy] #003 CONNECT v.douyin.com:443              # 短链解析
[proxy] #004 CONNECT www.iesdouyin.com:443         # 分享页元数据
[proxy] #005 CONNECT aweme.snssdk.com:443          # 无水印地址还原
[proxy] #006 CONNECT v95-sz-cold.douyinvod.com:443 # CDN 视频流
```

后台统计「经代理/直连 = 6/0」，即零直连。另测：好+坏代理并存时解析仍成功（失败转移生效）；仅坏代理 + 强制代理开启时返回明确报错而非直连（防 IP 泄露生效）。
